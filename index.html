Description

A dashboard that allows an administrator to search for a specific user and add them to a group (IT, Sales, Support). Also, the administrator is able to view all users in a specific group.

Overall

  • Search user by name

Challenge

Add a user to a specific group. When changed the user must only be found in the specific group. Their information must also change in the "all" group.

Site

Solution

Each button has a click event listener that passes the user's object data.

The user.group value is stored in the group constant.

The correct group_array is selected based on the group.

A map function runs through the group_array and removes the name which matches the user.first_name.

The user is pushed into the new group_array.

Code

										const group = user.group;
			              let group_array;
			              
			                 if (group === "IT") {
			                   group_array = this.state.it;
			                 } else if (group === "Sales") {
			                   group_array = this.state.sales;
			                 } else if (group === "Support") {
			                   group_array = this.state.support;
			                }
			              
			                group_array.map((user_old) => {
			                  if (user_old.first_name === user.first_name ) {
			                    let user_index = group_array.indexOf(user_old);
			                    group_array.splice(user_index,1);
			                  }
			                });

Step 1

Each filter, has an onChange EventListener that runs the checkboxAnswer function.

The checkbox's name, value, and checked status are placed in constants. Along with the object of arrays that is set in the initial state and an array of strings that will act as the function's filters.

A forEach loop runs through the filterProperties to match the chexbox's name with the filter. If both of them match and the checked status is true, the value of the checked box will be pushed into it's array. If they both match but the checked status is false, the last value in the array will be removed. This is done so the arrays do not have any duplicates.

Once the user clicks the SEARCH button, the dropdown_filter_arrays object is passed to the back-end through a POST request.

Code

										checkboxAnswer (event) {
										const checkboxName = event.target.name;
										const checkboxValue = event.target.value;
										const checkboxChecked = event.target.checked;
										const dropdown_filter_arrays = this.state.dropdown_filter_arrays;
										const filterProperties = ["techstars"];
											 filterProperties.forEach((filterBy) => {
											  if (filterBy === checkboxName && checkboxChecked){
												   if (filterBy === "techstars") {
												    dropdown_filter_arrays.techstars.push(checkboxValue);
												    this.setState({dropdown_filter_arrays});
												   }
											 } else if (filterBy === checkboxName && !checkboxChecked){
											   if (filterBy === "techstars"){
												    dropdown_filter_arrays.techstars.splice(-1,1);
												    this.setState({dropdown_filter_arrays});
											   }
											  }
											 })
										}

Description

Fuel helps you compare your goal calories vs your daily calories in three simple steps. The first step is to get your goal calories. This is calculated by collecting some general information. The second step is to get your daily calories. This is calculated by searching food items and placing them in a breakfast, lunch, or dinner box. The third step will give you a breakdown.

Overall

  • Provide a smooth user experience. One specific case where this is done is on the step 2 page. The option box changes views when a user clicks on the different tab buttons.
  • Provide a smooth user interface. This can be found on the goal and daily calories counter. When a user completes step 1 and/or 2, the counter is instantly updated. This is done through redux.
  • Make the application mobile friendly.

Please check the process page on a desktop.

Challenge

Be able to pass the calories data for the specific item clicked to calculate the daily calories.

Site

Solution

To solve this, I compiled the data into an array and passed the needed object on every onClick event in the results list.

The result from the API call is passed through the renderResultList function. Two map functions run to extract the data needed. This data is placed in two different constants. An empty object named results is also created.

A map function runs through the itemsArray array which builds the data for the results object.

The results object keys are placed in a constant in order to run a map function through it. This returns an array which is stored in the foodItems constant. This array will pass the object needed on every onClick event in the results list.

Code

		              renderResultList(listData) {
		                const itemsArray = listData.hits.map(item => item.fields.item_name.split(" - ")[0]);
		                const caloriesArray = listData.hits.map(item => item.fields.nf_calories);
		                const results = {};
		              
		                itemsArray.map((item,index) => {
		                  if(results.hasOwnProperty(item)){
		                  results[item].calories.push(caloriesArray[index]);
		                  } else {
		                  results[item] = {};
		                  results[item].calories = [];
		                  results[item].calories.push(caloriesArray[index]);
		                  }
		                });
		              
		                const resultsNames = Object.keys(results);
		                const foodItems = resultsNames.map((foodName,index) => {
		                  return {name: foodName, calories: results[foodName].calories[0]};
		                });
		              
		                return <div>
		                  <ul>
		                    {foodItems.map((item) => {
		                      const displayInfo = `${item.name} = ${item.calories}`;
		                        return <li key={displayInfo} onClick={()=> this.handleCheckboxChange(item)}>
		                          {displayInfo}
		                        </li>
		                    })}
		                  </ul>
		                </div>
		              }

Description

This was the second version of this application. You can find the first version here ( Code | Site ).

Overall

  • Create a full CRUD application using Angular
  • Apply Angular directives such as ngMessages and ngAnimate.

Please check the process page on a desktop.

Challenge

How to successfully be able to edit a specific post. Be able to pass information from the view, to the controller, to making an HTTP request, and back with a responds to the controller.

Site

Solution

1: View

The form tag has an ng-submit with the function editPost. This function will be passed on to the controller with two arguments ( $event and post.id). The post.id is the post id for that specific post.

Each input tag has an ng-model for a specific value. These values will be passed and placed in the $scope.views.userForm object in the controller once the submit input is clicked.

Code

		              <div>
		                 <h3>Edit Post</h3>
		                 <form action="#" name="editPostForm" ng-submit="editPost($event, post.id)" novalidate>
		                    <input  type="text" name="title" ng-model="views.userForm.title" placeholder="Title"/>
		                    <input  type="text" name="author" ng-model="views.userForm.author" placeholder="Author"/>
		                    <input  type="text" name="description" ng-model="views.userForm.description" placeholder="Description"/>
		                    <input  type="submit" value="Submit"/>
		                 </form>
		              </div>

2: Controller

The MainController controller is injected with $scope and firstService. The $scope.views.userForm object is created with empty values. Here is where the values from the editPostForm form will be placed.

The function editPost will pass the information from the $scope.views.userForm object and the post.id to the firstService factory. 

		              app.controller('MainController', ['$scope', 'firstService', function($scope,
		                firstService) {
		                $scope.views = {};
		                $scope.views.userForm = {
		                  title: "",
		                  author: "",
		                  description: "",
		                  votes: 0
		                };
		              
		                $scope.editPost = function(event, post.id) {
		                  event.preventDefault();
		                  firstService.edit($scope.views.userForm, post.id);
		                }
		              
		              }]);

3: Factory

The firstService factory is injected with $http. The edit function runs making a put request and passing the userForm object along with the post.id id. 

		              app.factory('firstService', function($http) {
		              
		                return {
		                  edit: function(userForm, post.id) {
		                    return $http.put('/api/posts/' + post.id, userForm);
		                  }
		              });

4: Route

Express, Router, and Knex are required.

The put request runs and a knex call is made to select all posts from the posts table that match the req.params.id which in this case will be the post.id. The title, author, description, and votes are updated with the information from the userForm object.

Lastly, a res.json(msg) is returned which will be passed to the MainController controller. 

		              var express = require('express'),
		                router = express.Router(),
		                knex = require('../db/knex');
		              
		              router.put('/:id', function(req, res) {
		                knex('posts').where({
		                  id: req.params.id
		                }).update({
		                  title: req.body.title,
		                  author: req.body.author,
		                  description: req.body.description,
		                  votes: 0
		                }).then(function(msg) {
		                  res.json(msg);
		                });
		              });
		              
		              module.exports = router;

5: View

The res.json(msg) is received and passed as an argument (msg).

The $scope.views.userForm object is cleared out and the getPosts function is triggered. This function will "reload" the posts so that the new edited information will show. 

		                $scope.editPost = function(event, id) {
		                  event.preventDefault();
		                  firstService.edit($scope.views.userForm, id).then(function(msg) {
		                    $scope.views.userForm = {
		                      title: "",
		                      author: "",
		                      description: "",
		                      votes: 0
		                    };
		                    $scope.getPosts();
		                  });
		                }
